Web Conference Taiwan / 2013 台灣網頁技術研討會

這個月 12 / 13 號台灣舉辦了首次以 Web 為主題的研討會 WebConf.tw (Web Conference Taiwan)。在開放報名到差不多沒票的時候我才聽說這個消息，曾經對網站技術頗有興趣的我還在猶豫要不要來報名的時候，很幸運的主辦單位同時也聯絡我詢問是否有意願替活動進行攝影記錄。只是當時我手邊剛整理完前一次婚禮攝影的照片，修圖地獄的慘狀仍歷歷在目，加上 Web Conference 為其兩天想必照片數量也會很多，使我當下着實猶豫了一會。

不過考慮過後覺得這是個相當難得的機會，不管是可以把議程聽到飽接觸這個領域裡的佼佼者，把所有到場的正妹通通拍一遍紮實的磨鍊一遍活動攝影的技術，或者是到場白吃白喝聊天打屁認識參與活動相關的工作人員，都是平常少有的嗆司，就決定趕快舉手報名以免好康被別人搶走接下這一次活動攝影記錄的任務。在活動結束了之後回顧這兩天拍攝的照片，打從心底著實感謝主辦人高見龍及 PCT，今天我才有機會在這裡寫這篇文章。



那麼就來進入正題吧。兩天的 Web Conference 共有數十個議程，大部分的議程會以三個一組同時的方式進行，平均約 30~40 分鐘一輪；而單獨的議程 ( keynote ) 則由領域相關重量級人物帶來 50 分鐘的講題，例如噗浪 (plurk.com) 的共同創辦人雲惟彬。看到這裡大約就能知道，一個議程平均包含趕場只有10分鐘左右可以拍攝，而在 Keynote 議程可以多花一些時間補齊一般議程無法拍攝的構圖。這兩天基本上除了 key note ，並沒有太多的時間休息，所以把握空檔喝喝水，上上廁所也就變得非常重要了；攝影師除了做好照片的品質管理之外，時間管理也是相當重要的。

關於攝影器材，經過考量這次決定攜帶包括 24mm f/1.4G、85mm f/1.4G、24-70mm f/2.8G、8mm f/3.5 Circular fisheye、105mm f/2.8G、50-500mm f/4.5-5.6 等共六顆鏡頭、兩盞閃燈以及一支腳架。大光圈鏡頭負責低光場合，標準焦段用來抓拍，魚眼以及望遠鏡則用來創作，105mm f/2.8 則是預防 85mm 焦段不夠而攜帶 – 其實更好的選擇會是 70-200mm f/2.8 ，不過至今我尚未購入這顆鏡頭。總之，鏡頭群預定是這樣的規劃。

事實上，由於主辦單位希望演講時不要打閃燈，加上室內照明有限，最後 105mm f/2.8G、50-500mm f/4.5-5.6、閃燈以及腳架最後是沒有使用到的。雖然室內照明色溫不錯，但由於投影布幕以及筆記型電腦螢幕的光線，白平衡為這次攝影的一個重點；加上演講時獎台上的燈是會關起來的，因此除了台上台下的白平衡會不同外，亮度也呈現三個級差 – 投影幕、講者、聽眾 三者的亮度均不同，白平衡也不同。複雜的白平衡為修圖帶來一些困難 – 你是要讓背景黃一些呢，還是要讓人臉藍一些？在比較極端的例子我會針對照片中不同的部分做白平衡的校正。

由下方左圖可看出來現場光線的狀況；有自螢幕打出的藍光、主講者頭上的黃光、還有這裡並沒有看到、照在講者臉上講台上的電腦打出的光線。下右圖原本投影布幕並非黑白，因為拍攝時反差過大，區域調整清晰度、對比度後相當的藍，再降低飽和度後便得到這樣的結果，雖然無法呈現當時的畫面，不過仍將主題與講者一同帶入了畫面。

即便光線及白平衡有些複雜，這次攝影屬於靜態活動，加上議程 ( 除了 Lightning talk ) 通常會有足夠的時間，所以可以多花一些時間推敲適當的設定；這一次室內拍攝大多使用 1/100秒快門、光圈 f/1.4~2.0 搭配感光度 1250~2000 拍攝，感謝可靠的機身在這樣的感光度下仍能輸出品質可接受的照片。當然，事後仍少不了軟體降噪等程序。

下圖左可看出實際上講台上的光線十分的昏暗。

場地本身也帶來不少拍攝上的限制，這並不是場地有缺陷，而是我相信不論任何情況或任何主題，攝影原本就會有許多問題需要克服。比方說，講台及座位上都有的麥克風常常阻礙構圖，使得拍攝講者臉部時常被擋住；講者同時也拿著無線麥克風，有時就將臉給擋住了；有一個場地同時有三個緊鄰的投影布幕(其中一個其實是電視牆)，前方還有一排桌子，加上大部分的講者都直接站在講台後方，要拍攝講者的全身照通常只能從後方或側面來拍攝了。

這位主講人在議程進行的同時也不停地在舞台前左右移動，雖然很難抓拍，不過避開了講台等障礙物(右邊黑黑糊糊那塊)，反而能拍出更有特色的照片。 ( 所以說看到攝影師在拍照時要記得使出輪擺式位移 (!?) )

善用構圖也是一個拍攝的重點。議程進行在同樣的場地，不斷不斷地重覆相似的事件，為了避免一下子就江郎才盡，不斷的跑位、嘗試不同的作法，可以產出更多具不同內容的照片。下面這張照片雖然曝光不足，但相當的有張力。 (其實如果曝光夠了，畫面大概也白了。) 不過不斷地移動會讓 stamina 體力消耗的很快，所以記得要帶幾罐 Stamina Potion 可以試著在移動前多思考，想要怎樣的效果，還有什麼樣的方式可以嘗試等等。

觀察到鋪浪創辦人 Alvin 的投影片有很多的大字報以後，我就開始守株待兔… 於是就拍到了，well … Alvin 在做下圖這張頭影片的時候心情可能不太好 …。 近年來高橋流投影片心法相當流行，這類型的投影片在關鍵時刻可以製造非常好的構圖。而這類型的敘事法相對於長篇大論平鋪直敘的演說方式來說更有情緒宣染的效果，所以這時別忘了也回頭看一下觀眾們的反應。


有些場次過多的人數也帶來一些困難，首先是佔掉了良好構圖的空間 ( 簡單地說就是好位置 – 這也是為什麼每到跨年在象山上都會鎖了一排腳架，因為他們要佔好位 )，再來是在移動的同時要不斷的「抱歉借過」、「excuse me」、「書咪媽線」然後好不容易擠出後門也已經又過了不少時間。不過，聽眾分佈的變化有時也是構圖靈感的來源；你擋住我的路，我就拍你！拍拍拍的照片不知不覺也就爆量了這樣，由於聽眾相當的多，據稱報名人數在 600 人以上，光就拍攝人像來說，每個人都拍一張就超過 600 了。所以我記得其中一個 keynote 議程中拍到無力的我終於攤在最前排的欄杆上，直接對著聽眾席上掃射，每一張都可以拍到不同的人，說起來還蠻痛快的，好像藍波拿著機關槍一樣。

由於大部分的時間都在攝影，加上每場只逗留十分多鐘，基本上議程的內容對我來說頂多是捕捉構圖瞬間的提示，沒辦法特別認真地聽講；不過過程多少還是會不小心專心聽了一小段議程的內容，簡單地做一點摘要：

“那些 Mockup 沒告訴你的事“: Adam Wang / 要小心你的按鈕太長！

“Maintainable CSS with Sass and Compass“: hlb / 用了 Compass 就變成美國隊長了

“你們都誤解了，網路是很安全的！“: Allen / 一不小心 W 就等於 VV了 (下左)
“網站分析？我小的時候覺得自己會“: BobChao / 有些名詞聽了這場就學到了，比方說 Bounce Rate (下右 – 猜猜螢幕上寫的是什麼？)

“用原型驅動設計“: Justin / 這位講者有參加 g0v hackath0n, 他這次也有談到他們當時的作品

“在家也能架出好網站 – 非典型精實貧窮網站維運方式分享“: Ryudo / 不考慮機架，因為太吵。可惜沒辦法聽完，我也想在家裡架站..XD (下左)
“建置 PHP 應用程式的秘密配方“: c9s / 好像曾經是同事 (下右)

“g0v 黑客松 – 寫程式改造社會“: Clkao / 寫程式改造政府，以 VCS 類比立法院，順便宣傳了 g0v Hackath1n，歡迎大家來參加。

因為我每一場都有跑所以大約比較得出來，印象中最爆滿的場次就是 hlb 及 Hitomi 的議程吧。
“溝通的藝術：設計與開發如何真心合作？“: Hitomi / 設計與開發如何真心合作？大家對這個問題都十分的好奇，其實我也是只是我每場都只有十分鐘可以待所以…

“錯失過的機會“: 雲惟彬 (Alvin) / 雖然演講過程充滿歡樂，但這個題目帶了一點淡淡的心酸。就連投影片封面都是藍色的。話說，他提到西方的社交網站相較於中國的獲利較差，主要在於西方只曉得從廣告中賺錢。由於噗浪先前也推出了廣告，那時候我還蠻想問實際上噗浪中廣告的效益如何的，不過最後我還是把 Q&A 時間留給別人了。

“(Hacker’s) Best Practice – The Upload“: Orange / 上傳圖檔夾資訊、路徑命名影響檔案類型判別、副檔名識別與順序等等.. 不愧是 (Hacker’s) Best Practice，翻了翻 IRC Log，大家對他都很愛戴阿！

大概就這樣吧。最後一共交出了千餘張的照片，兩天平均下來每一分鐘就產生一張照片。大型研討會的攝影記錄是件相當有趣的事，你可以看到各種人與人之間的互動，而且名正言順地拍下來；也讓你更仔細地注意到每個講者的小動作、習慣、以至於最美的角度或是最小的小腹。相對的，也可以認識一些朋友，比方說之前在 Hacking Thursday 碰過面的 Trista，Moztw 的極地狐，還有同樣負責活動攝影的 othree (雖然之前就見過了)、丞相及日落，還有活動主辦人 Eddie & PCT。雖然當下跟事後都花了很多力氣在照片上，未來有機會的話還是會想參與類似的活動。
